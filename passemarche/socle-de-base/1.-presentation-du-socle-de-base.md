# 1. Présentation du socle de base

Le **socle de base** correspond au référentiel de champs utilisés dans Passe Marché pour structurer les informations demandées et affichées dans les différents parcours, notamment côté acheteur et côté candidat.

Il constitue un point central d’administration permettant :

* d’organiser les informations selon une logique commune,
* de définir la manière dont chaque donnée est alimentée,
* d’assurer la cohérence entre les différents écrans et usages de la plateforme.

Le socle de base joue ainsi un rôle structurant dans la configuration des informations métier manipulées dans Passe Marché.\
\
Le socle de base permet d’administrer les champs qui composent les formulaires et vues de Passe Marché.

Pour chaque champ, il est possible de définir :

* sa nature,
* son caractère obligatoire ou non,
* son périmètre d’usage selon le type de marché,
* son mode d’alimentation (récupération automatique via API ou saisie manuelle),
* ainsi que son intitulé et sa description, côté acheteur comme côté candidat.

Cette organisation permet de disposer d’un cadre unique et maîtrisé pour piloter les informations présentées dans l’application.
