# 5. Consulter et modifier un champ existant

Chaque champ du socle peut être ouvert dans une page de détail permettant d’en consulter les paramètres et, selon les droits de l’utilisateur, d’en modifier les informations.\
\
<br>

<figure><img src="../.gitbook/assets/image (5).png" alt=""><figcaption></figcaption></figure>

La page de détail présente en en-tête :

* le nom du champ,
* les types de marché auxquels il s’applique,
* sa source,
* ainsi que l’action d’archivage si elle est autorisée.

L’administrateur peut ajuster :

* le type du champ,
* son caractère obligatoire,
* les types de marché associés.<br>

La description du champ est organisée en deux volets distincts :

* **vue acheteur**,
* **vue candidat**.

Chaque volet permet d’ajuster :

* la catégorie,
* la sous-catégorie,
* le titre,
* et la description.

La page permet également de choisir ou de modifier le mode d’alimentation du champ :

* API,
* ou manuel.

Lorsque la récupération via API est activée, la source correspondante doit être renseignée.

Un bouton permet d’enregistrer les modifications apportées.\
Un bouton d’annulation permet de revenir à l’état initial sans enregistrer les changements en cours.\
