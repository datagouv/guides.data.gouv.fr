# Outils

La liste d’outils suivante a pour objectif de répondre aux besoins des animateurs et participants aux groupes de travail lorsqu’aucun outil fourni par leur organisme d’appartenance ne convient. Le CNIG recommande d’avoir en priorité recours aux outils de l’Etat tels que ceux de [la suite numérique](https://lasuite.numerique.gouv.fr/). D’autres outils, libres d’usage ou payants sont également listés dans le cas ou aucun outil de l’Etat ne répondrait au besoin. Les outils sont listés par cas d’usage ci-dessous :

### <mark style="background-color:purple;">Outils classés par type de besoin</mark> <a href="#outils-classes-par-type-de-besoin" id="outils-classes-par-type-de-besoin"></a>

#### Gestion de projet : <a href="#gestion-de-projet" id="gestion-de-projet"></a>

* **Outils proposés par l’Etat :**
  * [Le Guide des communautés](https://www.modernisation.gouv.fr/files/2024-10/202410_Le_guide_des_communautes_mooc.pdf) fournit des recommandations pour un nouvel animateur de communauté ou de groupe de travail ;
* **Outils libres :**
  * [Kanboard](https://kanboard.org/) est un outil en ligne de type kanban ;

#### **Travail collaboratif :** <a href="#travail-collaboratif" id="travail-collaboratif"></a>

* **Outils proposés par l’Etat :**
  * [Tchap](https://www.tchap.gouv.fr/) pour la communication instantanée ;
  * [Docs](https://docs.numerique.gouv.fr/home/) pour la rédaction de documents partagés en ligne ;
  * [Grist ](https://grist.numerique.gouv.fr/)pour la création de tableurs, formulaires, etc. partagés en ligne ;
  * [Resana ](https://resana.numerique.gouv.fr/public/)pour le partage d’un espace collaboratif ;
  * [Notepad de l’Etat](https://pad.numerique.gouv.fr/) pour partager des notes au format Markdown ;
  * [Visio](https://visio.numerique.gouv.fr/) pour tenir des réunions en visioconférence ;
  * [FranceTransfert](https://francetransfert.numerique.gouv.fr/) pour le transfert de documents volumineux ;
  * [Enquêteur en ligne](https://spote.developpement-durable.gouv.fr/offre/enqueteur-en-ligne-du-pole-ministeriel-ecologie?debut_lesOffres=84) pour réaliser des formulaires en ligne (fondé sur LimeSurvey et réalisé par le ministère de la transition écologique) ;
  * [Sympa ](https://spote.developpement-durable.gouv.fr/offre/administrer-des-listes-de-diffusion-courriels?debut_lesOffres=96)pour la gestion de liste de diffusions (élaboré par le ministère de la culture) ;
* **Outils libres :**
  * Les outils libres développés par [Framasoft](https://framasoft.org/fr/) (sondages, questionnaires, etc.), et en particulier [Excalidraw](https://excalidraw.com/) pour l’organisation d’ateliers participatifs (murs de post-it numériques par exemple) ;
  * Les outils libres développés par [la Digitale](https://ladigitale.dev/) (tableaux blancs collaboratifs, murs multimédia, nuage de mots, etc.) ;
* **Autres outils :**
  * [Github](https://github.com/) pour le partage de documents (attention, le site ne supporte pas la gestion de version de documents binaires comme des documents word). Une [instance du ministère de la transition écologique ](https://spote.developpement-durable.gouv.fr/offre/gitlab-forge-logicielle-et-livraison-continue)existe ;
  * [Miro](https://miro.com/) pour l’organisation d’ateliers participatifs (murs de post-it numériques par exemple) ;
  * [Notion](https://www.notion.so/) pour la rédaction de compte rendus de réunion, la rédaction asynchrone, le suivi par des bases de données (de tâches, de contacts, etc.) ;
  * [eXo Platform ](https://www.exoplatform.com/fr/)pour l’organisation d’un espace collaboratif ;
  * [Google Workspace](https://workspace.google.com/intl/fr/) (docs, sheets, forms, etc.), dont l’utilisation est gratuite mais n’est pas recommandée en raison de son manque de souveraineté ;

#### Outils spécifiques à la conception de standards : <a href="#outils-specifiques-a-la-conception-de-standards" id="outils-specifiques-a-la-conception-de-standards"></a>

* **Outils proposés par l’Etat :**
  * [Papyrus](https://list.cea.fr/fr/page/papyrus/) du CEA-List pour la rédaction de schémas UML ;
  * [CSV-detective](https://github.com/datagouv/csv-detective) pour la détection des formats de données à partir d’un fichier tabulaire ;
* **Outils libres :**
  * [Pandoc](https://pandoc.org/) pour la conversion entre les formats markdown, word et pdf ;
  * [Notepad++](https://notepad-plus-plus.org/downloads/) et ses extensions ;
  * [DBeaver](https://dbeaver.io/) ;
  * [DB Browser](https://sqlitebrowser.org/) ;
  * [UMLetino](https://www.umletino.com/umletino.html) pour des besoins de modélisation sans export XML ;
* **Autres outils :**
  * [Entreprise Architect](https://sparxsystems.fr/products/ea/) pour la rédaction de schémas UML;
  * [Visual Studio Code](https://code.visualstudio.com/) ;

#### Test et validation : <a href="#test-et-validation" id="test-et-validation"></a>

* **Outils proposés par l’Etat :**
  * [Validata ](https://validata.fr/)pour vérifier la conformité d’un jeu de données à un schéma ;
  * [L’outil de validation de l’IGN ](https://demo-validator.ign.fr/#/)pour les documents d’urbanisme ;

#### Communication : <a href="#communication" id="communication"></a>

* **Canaux de communication recommandés :** site du CNIG, newsletter du CNIG, GéoRézo, GéoTribu, TeamOpenData, Forum des géocommuns, le site de l’Afigéo, communauté arcOpole, réseaux sociaux (Linkedin, etc.)
* **Outils proposés par l’Etat :**
  * [Grist](https://lasuite.numerique.gouv.fr/produits/grist) pour la gestion d’une base de contacts (newsletter, GT) ;
  * [Webinaire de l’Etat](https://webinaire.numerique.gouv.fr/home) pour tenir des séminaires en ligne ;
* **Autres outils :**
  * Les ressources et canaux de la [communauté arcOpole](https://www.arcopole.fr/documents).

#### Déploiement : <a href="#deploiement" id="deploiement"></a>

* **Outils proposés par l’Etat :**
  * Le site [schema.data.gouv.fr](http://schema.data.gouv.fr/) permet d’identifier les jeux de données respectant un schéma (ou un standard du CNIG référencé comme schéma) , de les dénombrer et d’évaluer leur périmètre. Un suivi du nombre de jeux de données conformes, du nombre d’organisations productrices de données, et du périmètre ainsi couvert peut permettre d’évaluer le déploiement du standard manuellement. Les outils manquent encore pour répondre à cet objectif.
* **Autres outils :**
  * [Gitbook](https://www.gitbook.com/) pour la rédaction d’une documentation ;
