# Réaliser une symbologie ou une charte graphique

La standardisation des données peut également passer par l’adoption d’une symbolisation commune dans certains cas. La création d’une **charte graphique** et d’une **symbologie** basée sur un dictionnaire de symboles, doit ainsi fournir les informations nécessaires pour produire des visualisations, et notamment des cartographies, facilitant l’interprétation des données.

**Ces recommandations peuvent notamment préciser :**

* la description littérale des symboles
* leur traduction informatique dans des formats standards (QML, SLD, LYRX, etc.)
* le format de production des visuels (svg par exemple) ;
* la différenciation des représentations selon les échelles de travail (les mêmes objets n’ont pas à être représentés de la même manière à tous les ordres de grandeur) ;
* les choix esthétiques que le groupe de travail souhaite proposer (ou imposer).

La réalisation de la symbologie est un travail qui peut prendre un temps important. Il est recommandé de travailler en sous-groupe sur ce sujet.

{% hint style="info" %}
**Note :** il peut être utile de publier la symbologie sous différents formats, notamment sous la forme d’un projet QGIS (voir les ressources de la BD TOPO plus bas), pour faciliter son import dans les SIG par les concepteurs de cartes, et sous un format lisible (pdf, site web, etc.) à des fins de documentation pour les utilisateurs des cartes.
{% endhint %}

Voici **quelques exemples** de chartes graphiques pouvant servir d’inspiration :

* [La symbologie du standard StaR-EAU](https://cnigfr.github.io/StaR-Eau/Symbologie/),
* [La symbolisation adoptée pour la dématérialisation des documents d’urbanisme](https://github.com/cnigfr/DDU-SG5-SYMBOLISATION),
* [Les recommandations de symbologie du standard Risques](https://github.com/cnigfr/Geostandards-risques-ppr/blob/master/standard/Document.md#recommandations-de-symbologie) (profil Plan de Prévention des Risques),
* [Les ressources de la BD TOPO](https://geoservices.ign.fr/documentation/donnees/vecteur/bdtopo) de l’IGN pour créer un projet QGIS avec une symbolisation prédéfinie (voir les ressources “[Projets cartographiques](https://geoservices.ign.fr/sites/default/files/2023-08/Projets_Carto_BDT_3-3.zip)”, “[Styles Géoserver](https://geoservices.ign.fr/sites/default/files/2023-02/SLD_bdtopov3_Geoserver.zip)”, et “[Projets cartographiques pour les produits vecteur - Manuel utilisateur](https://geoservices.ign.fr/sites/default/files/2021-07/Projets_carto.pdf)”),
* [Le registre des symboles](https://portal.dgiwg.org/public_dgiwg/portrayal/sdl.php) du Defence Geospatial Information Working Group (DGIWG).
